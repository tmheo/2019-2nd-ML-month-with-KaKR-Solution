# 아키텍처 개요

## 프로젝트 정보

**프로젝트명**: 2019 2nd ML month with KaKR - 주택 가격 예측 솔루션
**목표**: Kaggle 주택 가격 예측 경쟁에서 앙상블 기반 다단계 스태킹 모델로 최고 성능 달성
**기술 스택**: Python 3.x, Jupyter Notebook, pandas, scikit-learn, XGBoost, LightGBM, CatBoost, Keras/TensorFlow
**데이터**: Kaggle 주택 거래 데이터 (특성: 주거 정보, 지리적 좌표, 시장 가격)

## 설계 패턴

### 다단계 스태킹 (Multi-Stage Stacking)

**개념**: 여러 기초 모델(Base Learners)의 예측을 메타 모델(Meta-Learner)이 학습하여 최종 예측을 생성

**3단계 구조**:

1. **데이터 준비 단계**
   - 원본 데이터 로드 및 전처리
   - 지리적 특성 기반 이웃 정보 추출
   - 집계 통계로 메타 특성 생성

2. **Stage 1 (기초 모델 단계)**
   - 6~10개 다양한 기초 모델 학습
   - 각 모델이 5-fold CV 통해 OOF 예측 생성
   - OOF 예측을 새로운 특성 행렬로 변환

3. **Stage 2 (메타 모델 단계)**
   - Stage 1 OOF를 입력 특성으로 사용
   - 메타 모델 (XGBoost, 신경망 등) 학습
   - 최종 예측 생성

**장점**:
- 서로 다른 모델의 강점을 결합하여 일반화 성능 향상
- 과적합 감소 (OOF로 인한 정규화 효과)
- 모델 다양성으로 인한 낮은 오류율

### 지리적 특성 엔지니어링

**개념**: 부동산 위치 정보를 활용하여 예측력 높은 특성 창출

**방법**:
- **Haversine 거리**: GPS 좌표 간 실제 거리 계산
- **반경 기반 집계**: 1km, 3km, 5km 반경 내 이웃의 특성 통계
- **K-NN 기반**: 가장 가까운 K(5, 10, 20)개 이웃의 특성 집계
- **좌표 변환**: PCA 축소, K-means 클러스터링, 방위각 계산

**특징**: 부동산의 지리적 환경(이웃의 품질)을 숫자화하여 예측력 향상

## 시스템 경계

### 입력 경계
- **Raw Data**: train.csv (21,000+ 행, ~20개 특성), test.csv
- **특성**: 주거 정보(침실, 욕실, 면적), 지리 정보(lat, long), 시장 정보(가격, 날짜)

### 처리 경계
- **Notebook 기반 파이프라인**: 순차적 실행
- **기초 모델**: 6-10개 다양한 알고리즘
- **메타 모델**: 단일 또는 가중치 기반 앙상블

### 출력 경계
- **최종 예측**: 로그 스케일에서 원본 가격 스케일로 변환
- **평가 지표**: RMSE (Root Mean Squared Error)
- **제출 형식**: CSV (id, price)

## 핵심 모듈 관계

```
utils.py (모델 래퍼 + 헬퍼 함수)
├─ 모델 래퍼 클래스: XgbWrapper, LgbmWrapper, CatWrapper, KerasWrapper, ...
├─ 데이터 처리: load_data(), reduce_mem_usage()
├─ 지리 계산: haversine_array(), bearing_array()
├─ 앙상블 함수: get_oof(), stacking()
└─ 유틸리티: rmse(), time_decorator()

Notebook들이 utils.py를 import하여 사용
├─ geo-data-eda-and-feature-engineering.ipynb
│  └─ load_data() 호출
├─ Generate Neighbor Info.ipynb
│  └─ haversine_array(), bearing_array() 호출
├─ Generate Neighbor Stat.ipynb
│  └─ 집계 함수 직접 구현
└─ Stacking Ensemble.ipynb
   ├─ XgbWrapper, LgbmWrapper, ... 사용
   ├─ get_oof(), stacking() 호출
   └─ rmse() 평가
```

## 데이터 흐름 요약

1. **EDA 및 피처 엔지니어링**: 원본 데이터 → 100+ 개수치 특성
2. **지리적 이웃 추출**: 좌표 → 거리 기반/K-NN 이웃 정보
3. **통계 집계**: 이웃 정보 → 집계 특성 (mean, std, etc)
4. **Stage 1 학습**: 전체 특성 → 6-10개 모델 OOF
5. **메타 특성 생성**: Stage 1 OOF → 새로운 입력 특성
6. **Stage 2 학습**: 메타 특성 → 최종 메타 모델
7. **예측 변환**: 로그 스케일 → 원본 가격 스케일
